# Бізнес-шаблон Telegram-бота онлайн-запису клієнтів ("Записуха")

Готовий до використання, повністю локалізований Telegram-бот для онлайн-запису клієнтів у салони краси, до майстрів манікюру, косметологів, перукарів або незалежних спеціалістів. Бот підтримує динамічну мультиязичність (i18n), позмінний графік роботи (схема 2/2 між двома адресами: салон та вдома), перевірку вільних вікон для запису, автоматичні акційні знижки та інтеграцію з Google Sheets як основною базою даних (що гарантує збереження всіх даних при перезапусках хмарного сервера!).

## Основні переваги та функції

1. **Інтеграція з Google Sheets:** Жодних складних налаштувань баз даних SQL. Інформація про клієнтів, записи, прайс-листи, вихідні та заблоковані часи зберігається безпосередньо у вашій Google Таблиці.
2. **Динамічна мультиязичність (i18n):** Бот автоматично зчитує мову інтерфейсу Telegram користувача (українська, англійська, російська) та відповідає йому відповідною мовою.
3. **Графік роботи 2/2:** Бот автоматично визначає адресу прийому (наприклад, Салон або Вдома) за циклічною схемою 2/2, починаючи від заданої дати.
4. **Step-by-Step Бронювання (FSM):** Клієнт проходить зручний процес вибору дати, вільного часу, потрібної послуги, введення коментарів та фінального підтвердження.
5. **Адмін-панель у Telegram:** Адміністратор або майстер може переглядати записи на сьогодні та завтра, блокувати окремі години чи цілі дні безпосередньо через інтерфейс чату.
6. **Автоматичні акції:** Бот знаходить клієнтів, які давно не робили записи (наприклад, понад 21 день), та автоматично пропонує їм 10% знижку при бронюванні.

## Налаштування Google Sheets API

Оскільки бот використовує Google Таблицю для збереження інформації, вам необхідно створити сервісний акаунт Google та поділитися з ним таблицею.

### 1. Створення Сервісного акаунту Google
1. Перейдіть у [Google Cloud Console](https://console.cloud.google.com/).
2. Створіть новий проект (наприклад, `zapisukha-beauty-bot`).
3. Перейдіть у розділ **APIs & Services** -> **Library**. Знайдіть у пошуку **Google Sheets API** та **Google Drive API** та увімкніть (**Enable**) обидва API.
4. Перейдіть у **APIs & Services** -> **Credentials**. Натисніть **+ Create Credentials** -> **Service Account**.
5. Заповніть дані, натисніть **Create and Continue**, оберіть роль **Editor** (Редактор) для доступу та натисніть **Done**.
6. Знайдіть створений сервісний акаунт у списку, відкрийте його вкладку **Keys** (Ключі) -> **Add Key** -> **Create New Key**.
7. Оберіть формат **JSON** та натисніть **Create**. На ваш комп'ютер завантажиться файл ключа `.json`.
8. Перейменуйте завантажений файл на `credentials.json` та перенесіть його в кореневу папку бота.

### 2. Спільний доступ до таблиці
1. Створіть нову Google Таблицю у своєму Google Диску.
2. Відкрийте доступ до таблиці (кнопка **Share** / Поділитися).
3. Додайте поштову адресу сервісного акаунту (ви знайдете її у файлі `credentials.json` як `client_email`) та надайте їй права **Editor** (Редактор).
4. Скопіюйте унікальний **ID таблиці** з адресного рядка браузера:
   `https://docs.google.com/spreadsheets/d/YOUR_GOOGLE_SHEETS_ID/edit`
5. Збережіть цей ID в файл `.env` як `GOOGLE_SHEETS_ID`.

### 3. Структура Google Таблиці
Створіть листи у вашій Google Таблиці з такими назвами:
* **Клієнти** (Стовпчики: `ID`, `Ім'я`, `Телефон`, `Username`, `Нотатки`, `Дата`, `Статус`, `Крок`, `Фото`, `MsgID`)
* **Записи** (Стовпчики: `ID`, `Дата`, `Час`, `КлієнтID`, `Послуга`, `Статус`, `Побажання`, `Ціна`)
* **Вихідні** (Один стовпчик: `Дата`)
* **БлокЧас** (Стовпчики: `Дата`, `Час`)
* **Прайс** (Стовпчики: `Послуга`, `Ціна` - наприклад, рядок 1: `💅 Манікюр` | `580`)

## Встановлення та локальний запуск

### 1. Telegram Токен
1. Знайдіть у Telegram бот [@BotFather](https://t.me/BotFather) та створіть нового бота за допомогою команди `/newbot`.
2. Скопіюйте та збережіть `BOT_TOKEN`.
3. У налаштуваннях бота в BotFather задайте команди меню:
   - `start` - Головне меню
   - `admin` - Панель адміністратора

### 2. Файл налаштувань (.env)
Створіть файл `.env` в кореневій папці проекту (скопіюйте вміст з `.env.example`) та заповніть параметри:
```ini
BOT_TOKEN=ВАШ_TELEGRAM_BOT_TOKEN
ADMIN_IDS=ВАШ_TELEGRAM_USER_ID

GOOGLE_CREDENTIALS_FILE=credentials.json
GOOGLE_SHEETS_ID=ВАШ_ID_GOOGLE_ТАБЛИЦІ

# Адреси локацій
ADDRESS_SALON=📍 вул. Шевченка 128 (Салон)
ADDRESS_HOME=🏠 вул. Сонячна 15 (Дім)

SHIFT_START_DATE=2026-04-21
WORK_HOURS=10:00,12:00,14:00,16:00
```

### 3. Локальний запуск
1. Активація віртуального оточення:
   ```bash
   python -m venv venv
   # На Windows (cmd):
   venv\Scripts\activate
   # На macOS/Linux:
   source venv/bin/activate
   ```
2. Встановлення залежностей:
   ```bash
   pip install -r requirements.txt
   ```
3. Запуск бота:
   ```bash
   python main.py
   ```

## Налаштування Git та GitHub (Критично важливо для деплою)

Перед розгортанням бота на хмарних хостингах (наприклад, Railway), вам потрібно викласти код у репозиторій Git.

> [!WARNING]
> **Завжди створюйте ПРИВАТНИЙ репозиторій на GitHub.** Якщо зробити репозиторій публічним, то це відкриє ваш файл конфігурації `.env`, секретний файл авторизації Google API (`credentials.json`) та токен бота для всього світу. 

1. **Ініціалізуйте Git локально:**
   ```bash
   git init
   git add .
   git commit -m "Початковий коміт бота онлайн-запису"
   ```
2. **Створіть приватний репозиторій на GitHub:**
   - Перейдіть на [GitHub](https://github.com/) та створіть новий репозиторій.
   - Обов'язково виберіть опцію **Private** (Приватний).
   - **Не** ініціалізуйте репозиторій файлами README, `.gitignore` або ліцензією (вони вже є в проекті).
3. **Прив'яжіть та відправте локальний код на GitHub:**
   ```bash
   git remote add origin https://github.com/ВАШ_ЛОГІН/ІМ'Я_РЕПОЗИТОРІЮ.git
   git branch -M main
   git push -u origin main
   ```

## Деплой на сервери (Railway / VPS)

Цей шаблон містить файл **Procfile** та повністю підготовлений до розгортання на серверах Railway:
1. Зареєструйтесь на [Railway](https://railway.app/).
2. Створіть **New Project** -> **Deploy from GitHub repo**.
3. Оберіть ваш **Приватний** репозиторій з ботом.
4. У вкладці **Variables** вашого сервісу на Railway додайте всі змінні з локального файлу `.env` (`BOT_TOKEN`, `GOOGLE_SHEETS_ID`, `ADMIN_IDS` тощо).
5. Оскільки репозиторій є приватним, файл авторизації Google API (`credentials.json`) завантажиться автоматично разом із кодом, тому бот відразу буде готовий до роботи.

Railway самостійно збере оточення та запустить бота. Усі записи зберігаються у хмарній таблиці Google Sheets, тому дані клієнтів ніколи не зникнуть при оновленні контейнера або перезапусках серверів!

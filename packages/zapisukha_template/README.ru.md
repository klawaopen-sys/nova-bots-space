# Бизнес-шаблон Telegram-бота онлайн-записи клиентов ("Записуха")

Готовый к развертыванию, полностью локализованный Telegram-бот для онлайн-записи клиентов в салоны красоты, к мастерам маникюра, косметологам, парикмахерам или независимым специалистам. Бот поддерживает динамическую мультиязычность (i18n), посменный график работы (схема 2/2 между двумя адресами: салон и на дому), проверку свободных окон для записи, автоматические акционные скидки и интеграцию с Google Sheets в качестве основной базы данных (что гарантирует сохранность всех данных при перезапусках облачного сервера!).

## Основные преимущества и функции

1. **Интеграция с Google Sheets:** Никаких сложных настроек баз данных SQL. Информация о клиентах, записях, прайс-листах, выходных и заблокированных часах хранится напрямую в вашей Google Таблице.
2. **Динамическая мультиязычность (i18n):** Бот автоматически считывает язык интерфейса Telegram пользователя (украинский, английский, русский) и отвечает ему на соответствующем языке.
3. **График работы 2/2:** Бот автоматически определяет адрес приема (например, Салон или На дому) по циклической схеме 2/2, начиная от заданной даты.
4. **Step-by-Step Бронирование (FSM):** Клиент проходит удобный процесс выбора даты, свободного времени, нужной услуги, ввода комментариев и финального подтверждения.
5. **Админ-панель в Telegram:** Администратор или мастер может просматривать записи на сегодня и завтра, блокировать отдельные часы или целые дни непосредственно через интерфейс чата.
6. **Автоматические акции:** Бот находит клиентов, которые давно не делали записи (например, более 21 дня), и автоматически предлагает им 10% скидку при бронировании.

## Настройка Google Sheets API

Поскольку бот использует Google Таблицу для сохранения информации, вам необходимо создать сервисный аккаунт Google и поделиться с ним таблицей.

### 1. Создание Сервисного аккаунта Google
1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/).
2. Создайте новый проект (например, `zapisukha-beauty-bot`).
3. Перейдите в раздел **APIs & Services** -> **Library**. Найдите в поиске **Google Sheets API** и **Google Drive API** и включите (**Enable**) оба API.
4. Перейдите в **APIs & Services** -> **Credentials**. Нажмите **+ Create Credentials** -> **Service Account**.
5. Заполните данные, нажмите **Create and Continue**, выберите роль **Editor** (Редактор) для доступа и нажмите **Done**.
6. Найдите созданный сервисный аккаунт в списке, откройте его вкладку **Keys** (Ключи) -> **Add Key** -> **Create New Key**.
7. Выберите формат **JSON** и нажмите **Create**. На ваш компьютер загрузится файл ключа `.json`.
8. Переименуйте загруженный файл в `credentials.json` и перенесите его в корневую папку бота.

### 2. Совместный доступ к таблице
1. Создайте новую Google Таблицу на своем Google Диске.
2. Откройте доступ к таблице (кнопка **Share** / Поделиться).
3. Добавьте почтовый адрес сервисного аккаунта (вы найдете его в файле `credentials.json` как `client_email`) и предоставьте ей права **Editor** (Редактор).
4. Скопируйте уникальный **ID таблицы** из адресной строки браузера:
   `https://docs.google.com/spreadsheets/d/YOUR_GOOGLE_SHEETS_ID/edit`
5. Сохраните этот ID в файл `.env` как `GOOGLE_SHEETS_ID`.

### 3. Структура Google Таблицы
Создайте листы в вашей Google Таблице со следующими названиями:
* **Клієнти** (Столбцы: `ID`, `Ім'я`, `Телефон`, `Username`, `Нотатки`, `Дата`, `Статус`, `Крок`, `Фото`, `MsgID`)
* **Записи** (Столбцы: `ID`, `Дата`, `Час`, `КлієнтID`, `Послуга`, `Статус`, `Побажання`, `Ціна`)
* **Вихідні** (Один столбец: `Дата`)
* **БлокЧас** (Столбцы: `Дата`, `Час`)
* **Прайс** (Столбцы: `Послуга`, `Ціна` - например, строка 1: `💅 Манікюр` | `580`)

## Установка и локальный запуск

### 1. Telegram Токен
1. Найдите в Telegram бот [@BotFather](https://t.me/BotFather) и создайте нового бота с помощью команды `/newbot`.
2. Скопируйте и сохраните `BOT_TOKEN`.
3. В настройках бота в BotFather задайте команды меню:
   - `start` - Главное меню
   - `admin` - Панель администратора

### 2. Файл настроек (.env)
Создайте файл `.env` в корневой папке проекта (скопируйте содержимое из `.env.example`) и заполните параметры:
```ini
BOT_TOKEN=ВАШ_TELEGRAM_BOT_TOKEN
ADMIN_IDS=ВАШ_TELEGRAM_USER_ID

GOOGLE_CREDENTIALS_FILE=credentials.json
GOOGLE_SHEETS_ID=ВАШ_ID_GOOGLE_ТАБЛИЦЫ

# Адреса локаций
ADDRESS_SALON=📍 ул. Шевченко 128 (Салон)
ADDRESS_HOME=🏠 ул. Солнечная 15 (Дом)

SHIFT_START_DATE=2026-04-21
WORK_HOURS=10:00,12:00,14:00,16:00
```

### 3. Локальный запуск
1. Активация виртуального окружения:
   ```bash
   python -m venv venv
   # На Windows (cmd):
   venv\Scripts\activate
   # На macOS/Linux:
   source venv/bin/activate
   ```
2. Установка зависимостей:
   ```bash
   pip install -r requirements.txt
   ```
3. Запуск бота:
   ```bash
   python main.py
   ```

## Настройка Git и GitHub (Критически важно для деплоя)

Перед развертыванием бота на облачных хостингах (например, Railway), вам нужно выложить код в репозиторий Git.

> [!WARNING]
> **Всегда создавайте ПРИВАТНЫЙ репозиторий на GitHub.** Если сделать репозиторий публичным, то это откроет ваш файл конфигурации `.env`, секретный файл авторизации Google API (`credentials.json`) и токен бота всему миру. 

1. **Инициализируйте Git локально:**
   ```bash
   git init
   git add .
   git commit -m "Начальный коммит бота онлайн-записи"
   ```
2. **Создайте приватный репозиторий на GitHub:**
   - Перейдите на [GitHub](https://github.com/) и создайте новый репозиторий.
   - Обязательно выберите опцию **Private** (Приватный).
   - **Не** инициализируйте репозиторий файлами README, `.gitignore` или лицензией (они уже есть в проекте).
3. **Привяжите и отправьте локальный код на GitHub:**
   ```bash
   git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
   git branch -M main
   git push -u origin main
   ```

## Деплой на серверы (Railway / VPS)

Этот шаблон содержит файл **Procfile** и полностью подготовлен к развертыванию на серверах Railway:
1. Зарегистрируйтесь на [Railway](https://railway.app/).
2. Создайте **New Project** -> **Deploy from GitHub repo**.
3. Выберите ваш **Приватный** репозиторий с ботом.
4. Во вкладке **Variables** вашего сервиса на Railway добавьте все переменные из локального файла `.env` (`BOT_TOKEN`, `GOOGLE_SHEETS_ID`, `ADMIN_IDS` и т.д.).
5. Поскольку репозиторий является приватным, файл авторизации Google API (`credentials.json`) загрузится автоматически вместе с кодом, поэтому бот сразу будет готов к работе.

Railway самостоятельно соберет окружение и запустит бота. Все записи сохраняются в облачной таблице Google Sheets, поэтому данные клиентов никогда не пропадут при обновлении контейнера или перезапусках серверов!

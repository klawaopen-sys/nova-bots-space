# Конструктор локальних маркетплейсів Telegram (Барахолка у кишені)

Готовий до використання, масштабований та повністю автоматизований Telegram-бот для створення міських дошок оголошень. Бот оснащений ШІ-модерацією зображень через Gemini, голосовим диктуванням через Groq Whisper, динамічним визначенням мови користувача (i18n), реферальною програмою та оплатою лімітів через Telegram Stars.

## Основні переваги та функції

1. **Кастомізація під будь-яке місто (White-Label):** Ви можете запустити локальну барахолку для *будь-якого* міста світу (Київ, Берлін, Варшава, Ніжин, Лондон тощо). Просто змініть змінну `CITY_NAME` у файлі `.env` — і бот автоматично налаштує під потрібне місто файли бази даних, привітання в меню, тексти кнопок та описи.
2. **Динамічна мультиязичність (i18n):** Бот автоматично зчитує мову інтерфейсу Telegram користувача (українська, англійська, російська) та відповідає йому відповідною мовою.
3. **ШІ-Модерація фото (Google Gemini):** Автоматично аналізує завантажені фотографії оголошень за допомогою моделі `gemini-2.5-flash`. Якщо користувач намагається завантажити картинку, що не відповідає товару (сміття, меми, неприйнятний вміст), бот відхилить оголошення.
4. **Голосове введення (Groq Whisper):** Користувачі можуть надиктовувати опис, назву або ціну голосом. Бот транскрибує повідомлення за допомогою Groq `whisper-large-v3-turbo` і заповнює відповідні текстові поля.
5. **Ліміти та інтеграція Telegram Stars:** Бот має тижневий ліміт безкоштовних оголошень (за замовчуванням 3). Користувач може придбати тижневий або місячний безліміт за зірки Telegram Stars (XTR).
6. **Багаторівнева реферальна програма:** Стимулює органічне зростання. Запрошувач отримує додаткові безкоштовні публікації, коли його реферал публікує перше оголошення (бонуси: Рівень 1 = +3, Рівень 2 = +2, Рівень 3 = +1).

## Інструкція зі встановлення та налаштування

### 1. Створення Telegram-бота
1. Перейдіть до боту [@BotFather](https://t.me/BotFather) у Telegram.
2. Надішліть команду `/newbot`, оберіть ім'я та юзернейм для вашого бота.
3. Скопіюйте наданий `BOT_TOKEN` та збережіть його.

### 2. Отримання API-ключів
- **Google Gemini API Key:** Створіть безкоштовний ключ у [Google AI Studio Console](https://aistudio.google.com/) для роботи модерації фото.
- **Groq API Key:** Зареєструйтесь на [Groq Console](https://console.groq.com/) та згенеруйте API-ключ для розпізнавання голосу.

### 3. Налаштування файлу оточення (.env)
Створіть файл `.env` у кореневій папці проекту (скопіюйте вміст з `.env.example`) та вкажіть власні налаштування:

```ini
# Токен бота від BotFather
BOT_TOKEN=ВАШ_TELEGRAM_BOT_TOKEN

# Ваш числовий ID користувача Telegram (для доступу до адмінки)
ADMIN_ID=ВАШ_TELEGRAM_USER_ID

# Локалізація та місто
CITY_NAME=Київ      # Назва міста для маркетплейсу
DEFAULT_LANG=uk    # Мова за замовчуванням (uk, en, ru)

# API Ключі
GEMINI_API_KEY=ВАШ_GEMINI_API_KEY
GROQ_API_KEY=ВАШ_GROQ_API_KEY
GROQ_MODEL=whisper-large-v3-turbo

# Налаштування проксі (опціонально)
USE_PROXY=False
```

### 4. Локальний запуск (тестування)
1. Переконайтеся, що у вас встановлено Python 3.10 або вище.
2. Створіть та активуйте віртуальне оточення:
   ```bash
   python -m venv venv
   # На Windows (cmd):
   venv\Scripts\activate
   # На macOS/Linux:
   source venv/bin/activate
   ```
3. Встановіть залежності:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустіть бота:
   ```bash
   python main.py
   ```

### 5. Налаштування Git та GitHub (Критично важливо для деплою)
Перед розгортанням бота на хмарних сервісах (наприклад, Railway), вам потрібно викласти код у репозиторій Git.

> [!WARNING]
> **Завжди створюйте ПРИВАТНИЙ репозиторій на GitHub.** Якщо зробити репозиторій публічним, то це відкриє ваш файл конфігурації `.env`, токен бота та приватні API-ключі для всього світу. Файл `.gitignore`, що входить до шаблону, запобігає коміту локальних файлів `.env` та баз даних `.db`, проте будьте максимально обережні.

1. **Ініціалізуйте локальний Git-репозиторій:**
   ```bash
   git init
   git add .
   git commit -m "Початковий коміт шаблону маркетплейсу"
   ```
2. **Створіть приватний репозиторій на GitHub:**
   - Перейдіть на [GitHub](https://github.com/) та створіть новий репозиторій.
   - Обов'язково виберіть опцію **Private** (Приватний).
   - **Не** ініціалізуйте репозиторій файлами README, `.gitignore` або ліцензією (вони вже є в проекті).
3. **Прив'яжіть та відправте локальний код на GitHub:**
   ```bash
   git remote add origin https://github.com/ВАШ_ЛОГІН/ІМ'Я_РЕПОЗИТОРІЮ.git
   git branch -M main
   git push -u origin main
   ```

### 6. Деплой на сервери (Railway / VPS)
Цей шаблон повністю підготовлений для безперервної інтеграції та автоматичного оновлення.

- **Деплой на Railway:**
  1. Створіть акаунт та авторизуйтесь на [Railway](https://railway.app/).
  2. Натисніть **New Project** -> **Deploy from GitHub repo**.
  3. Виберіть ваш щойно створений **Приватний** репозиторій.
  4. Перейдіть у вкладку **Variables** в налаштуваннях сервісу на Railway та додайте туди всі змінні з вашого файлу `.env` (наприклад, `BOT_TOKEN`, `ADMIN_ID`, `CITY_NAME`, `GEMINI_API_KEY`, `GROQ_API_KEY` тощо).
  5. Railway автоматично збере оточення за файлом `requirements.txt` та запустить бота за допомогою команди у `Procfile`.
- **Постійне зберігання бази даних (Railway Persistence):**
  - За замовчуванням файлова система Railway є ефемерною (скидається при кожному оновленні або перезапуску бота). Щоб дані оголошень не зникали:
    1. У панелі Railway натисніть кнопку **+ New** -> **Volume** (для створення диска).
    2. Примонтуйте створений Volume у будь-яку папку, наприклад `/data`.
    3. Додайте у **Variables** бота змінну `DB_NAME` зі значенням `/data/ads.db` (або будь-яким іншим ім'ям), щоб база даних SQLite записувалася безпосередньо на постійний диск.

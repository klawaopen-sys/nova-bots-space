# Конструктор локальных маркетплейсов Telegram (Барахолка в кармане)

Готовый к развертыванию, масштабируемый и полностью автоматизированный Telegram-бот для создания городских досок объявлений. Бот оснащен ИИ-модерацией изображений через Gemini, голосовым диктованием через Groq Whisper, динамическим определением языка пользователя (i18n), реферальной программой и оплатой лимитов через Telegram Stars.

## Основные преимущества и функции

1. **Кастомизация под любой город (White-Label):** Вы можете запустить локальную барахолку для *любого* города мира (Киев, Берлин, Варшава, Нежин, Лондон и др.). Просто измените переменную `CITY_NAME` в файле `.env` — и бот автоматически настроит под нужный город файлы базы данных, приветствия в меню, тексты кнопок и описания.
2. **Динамическая мультиязычность (i18n):** Бот автоматически считывает язык интерфейса Telegram пользователя (украинский, английский, русский) и отвечает ему на соответствующем языке.
3. **ИИ-Модерация фото (Google Gemini):** Автоматически анализирует загруженные фотографии объявлений с помощью модели `gemini-2.5-flash`. Если пользователь пытается загрузить неподходящую картинку (мусор, мемы, неприемлемый контент), бот отклонит объявление.
4. **Голосовой ввод (Groq Whisper):** Пользователи могут надиктовывать описание, название или цену голосом. Бот транскрибирует сообщение с помощью Groq `whisper-large-v3-turbo` и заполнит соответствующие текстовые поля.
5. **Лимиты и интеграция Telegram Stars:** Бот имеет недельный лимит бесплатных объявлений (по умолчанию 3). Пользователь может приобрести недельный или месячный безлимит за звезды Telegram Stars (XTR).
6. **Многоуровневая реферальная программа:** Стимулирует органический рост. Пригласивший получает дополнительные бесплатные публикации, когда его реферал публикует первое объявление (бонусы: Уровень 1 = +3, Уровень 2 = +2, Уровень 3 = +1).

## Инструкция по установке и настройке

### 1. Создание Telegram-бота
1. Перейдите к боту [@BotFather](https://t.me/BotFather) в Telegram.
2. Отправьте команду `/newbot`, выберите имя и юзернейм для вашего бота.
3. Скопируйте предоставленный `BOT_TOKEN` и сохраните его.

### 2. Получение API-ключей
- **Google Gemini API Key:** Создайте бесплатный ключ в [Google AI Studio Console](https://aistudio.google.com/) для работы модерации фото.
- **Groq API Key:** Зарегистрируйтесь на [Groq Console](https://console.groq.com/) и сгенерируйте API-ключ для распознавания голоса.

### 3. Настройка файла окружения (.env)
Создайте файл `.env` в корневой папке проекта (скопируйте содержимое из `.env.example`) и укажите собственные настройки:

```ini
# Токен бота от BotFather
BOT_TOKEN=ВАШ_TELEGRAM_BOT_TOKEN

# Ваш числовой ID пользователя Telegram (для доступа к админке)
ADMIN_ID=ВАШ_TELEGRAM_USER_ID

# Локализация и город
CITY_NAME=Киев      # Название города для маркетплейса
DEFAULT_LANG=ru    # Язык по умолчанию (ru, uk, en)

# API Ключи
GEMINI_API_KEY=ВАШ_GEMINI_API_KEY
GROQ_API_KEY=ВАШ_GROQ_API_KEY
GROQ_MODEL=whisper-large-v3-turbo

# Настройки прокси (опционально)
USE_PROXY=False
```

### 4. Локальный запуск (тестирование)
1. Убедитесь, что у вас установлен Python 3.10 или выше.
2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   # На Windows (cmd):
   venv\Scripts\activate
   # На macOS/Linux:
   source venv/bin/activate
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустите бота:
   ```bash
   python main.py
   ```

### 5. Настройка Git и GitHub (Критически важно для деплоя)
Перед развертыванием бота на облачных сервисах (например, Railway), вам нужно выложить код в репозиторий Git.

> [!WARNING]
> **Всегда создавайте ПРИВАТНЫЙ репозиторий на GitHub.** Сделать репозиторий публичным — значит открыть ваш конфигурационный файл `.env`, токен бота и приватные API-ключи всему миру. Файл `.gitignore`, включенный в шаблон, предотвратит коммит локальных файлов `.env` и баз данных `.db`, однако соблюдайте максимальную осторожность.

1. **Инициализируйте локальный Git-репозиторий:**
   ```bash
   git init
   git add .
   git commit -m "Начальный коммит шаблона маркетплейса"
   ```
2. **Создайте приватный репозиторий на GitHub:**
   - Перейдите на [GitHub](https://github.com/) и создайте новый репозиторий.
   - Обязательно выберите опцию **Private** (Приватный).
   - **Не** инициализируйте репозиторий файлами README, `.gitignore` или лицензией (они уже есть в проекте).
3. **Привяжите и отправьте локальный код на GitHub:**
   ```bash
   git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ.git
   git branch -M main
   git push -u origin main
   ```

### 6. Деплой на серверы (Railway / VPS)
Этот шаблон полностью подготовлен для непрерывной интеграции и автоматического обновления.

- **Деплой на Railway:**
  1. Создайте аккаунт и авторизуйтесь на [Railway](https://railway.app/).
  2. Нажмите **New Project** -> **Deploy from GitHub repo**.
  3. Выберите ваш только что созданный **Приватный** репозиторий.
  4. Перейдите во вкладку **Variables** в настройках сервиса на Railway и добавьте туда все переменные из вашего файла `.env` (например, `BOT_TOKEN`, `ADMIN_ID`, `CITY_NAME`, `GEMINI_API_KEY`, `GROQ_API_KEY` и т.д.).
  5. Railway автоматически соберет окружение по файлу `requirements.txt` и запустит бота с помощью команды в `Procfile`.
- **Постоянное хранение базы данных (Railway Persistence):**
  - По умолчанию файловая система Railway является эфемерной (сбрасывается при каждом обновлении или перезапуске бота). Чтобы данные объявлений не пропадали:
    1. В панели Railway нажмите кнопку **+ New** -> **Volume** (для создания диска).
    2. Примонтируйте созданный Volume в любую папку, например `/data`.
    3. Добавьте в **Variables** бота переменную `DB_NAME` со значением `/data/ads.db` (или любым другим именем), чтобы SQLite база записывалась напрямую на постоянный диск.
